# pico-adc-read — NAM-on-PWM Branch (`nam-pwm`)

Neural Amp Modeler A2-Lite running on Pico 2 (RP2350) with effects.
ADC on GP26 → DC blocker → NAM → reverb → PWM on GP19 at 200 kHz carrier.
No USB audio. USB CDC kept only for serial + picotool reboot.

## Flash (no unplugging)

```bash
picotool reboot -f -u && sleep 1 && picotool load build/pico_adc_read.uf2 && picotool reboot
```

If that fails, hold BOOTSEL while plugging in, then:
```bash
picotool load build/pico_adc_read.uf2 && picotool reboot
```

## Build

```bash
# Pico 2 (RP2350) only — dual Cortex-M33 at 300 MHz required
mkdir -p build && cd build && cmake .. -DCMAKE_BUILD_TYPE=Release && make -j$(sysctl -n hw.ncpu)

# With a custom model
cmake .. -DNAM_MODEL=my_tone.nam -DCMAKE_BUILD_TYPE=Release && make -j$(sysctl -n hw.ncpu)
```

## Hardware

- **GP19 (pin 25)** — PWM audio output, 200 kHz carrier
- **GP26 (pin 31)** — ADC audio input (LM358 preamp, AC-coupled, biased to ~1.65V)
- RC low-pass filter on GP19: e.g. 1 kΩ series + 10 nF to GND

## Runtime

| Mode | LED | Signal path |
|------|-----|-------------|
| NAM active | Solid on | ADC → DC blocker → NAM A2-Lite → Schroeder reverb → PWM |
| Passthrough | Slow blink (1 Hz) | ADC → PWM (direct) |
| 440 Hz tone | Fast blink (4 Hz) | Square wave → PWM |

Press & release **BOOTSEL** to cycle modes.

## Signal chain & gain staging

```
GP26 → LM358 preamp → ADC (12-bit, 48 kHz)
  → adc_to_float (bias-remove, ×0.01 input gain)
  → DC blocker (10 Hz high-pass, 1st-order IIR)
  → NAM A2-Lite (dual-core a2_fast, 300 MHz)
  → Schroeder reverb (4 combs + 2 allpasses, 35% wet)
  → float_to_pwm (soft-clip at ±0.95)
  → GP19 PWM (200 kHz carrier)
```

Key tunables in `src/main.cpp`:
- `INPUT_GAIN` (0.01) — scales ADC float before NAM
- `NAM_OUTPUT_GAIN` (0.5, in `src/nam_fx.cpp`) — scales NAM output
- `schroeder_set_mix()` (0.35) — reverb wet/dry

## Model

Current model: **EVH 5150 III 50W Stealth Red + Merciless Drive** (from [tone3000.com](https://tone3000.com))

To swap models from [tone3000.com](https://tone3000.com):

```bash
# Download any A2 capture, then:
cmake -S . -B build -DNAM_MODEL=my_tone.nam -DCMAKE_BUILD_TYPE=Release
cmake --build build -j
```

Only A2-Lite (3-channel) models are supported — `nam2c` validates at build time.
All A2-Lite models are 1,871 weights (~7.5 KB) — swapping is free.

## Adding effects

Effects plug into the block-processing loop in `src/main.cpp` (NAM mode only —
passthrough and tone bypass everything). To add a new effect:

1. Create `src/my_effect.h` with a C-linkage interface:
   ```c
   void my_effect_init(void);
   void my_effect_process(float* buf, int n);  // in-place
   ```
2. Implement in `src/my_effect.cpp`.
3. Add `src/my_effect.cpp` to `CMakeLists.txt` in the `add_executable` list.
4. Include the header, call `my_effect_init()` after `dc_blocker_init()` in `main()`.
5. Call `my_effect_process()` at the desired point in the chain (line ~290).

Current chain (NAM mode):
```c
dc_blocker_process(adc_in_buf[buf_idx], NFRAMES);  // after ADC
nam_fx_process(out_float, adc_in_buf[buf_idx], NFRAMES);
schroeder_process(out_float, NFRAMES);  // after NAM
ring_push(out_float);
```

## USB CDC notes

- `pico_enable_stdio_usb(target 1)` in CMakeLists.txt — all that's needed
- **Do not add a custom `tusb_config.h`** — the SDK provides its own
- **Do not add `usb_descriptors.c`** — the SDK's `stdio_usb_descriptors.c` handles it
