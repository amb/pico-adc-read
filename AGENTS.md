# pico-adc-read — NAM-on-PWM Branch (`nam-pwm`)

Neural Amp Modeler A2-Lite running on Pico 2 (RP2350).
ADC on GP26 → dual-core NAM → PWM on GP19 at 200 kHz carrier. No USB audio.
USB CDC kept only for serial + picotool reboot.

## Flash (no unplugging)

```bash
picotool reboot -f -u && sleep 1 && picotool load build/pico_adc_read.uf2 && picotool reboot
```

If that fails, hold BOOTSEL while plugging in, then:
```bash
picotool load build/pico_adc_read.uf2 && picotool reboot
```

## Build

```bash
# Pico 2 (RP2350) only — dual Cortex-M33 at 300 MHz required
mkdir -p build && cd build && cmake .. -DCMAKE_BUILD_TYPE=Release && make -j$(sysctl -n hw.ncpu)

# With a custom model
cmake .. -DNAM_MODEL=my_tone.nam -DCMAKE_BUILD_TYPE=Release && make -j$(sysctl -n hw.ncpu)
```

## Hardware

- **GP19 (pin 25)** — PWM audio output, 200 kHz carrier
- **GP26 (pin 31)** — ADC audio input
- RC low-pass filter on GP19: e.g. 1 kΩ series + 10 nF to GND

## Runtime

| Mode | LED | What it does |
|------|-----|--------------|
| NAM active | Solid on | Guitar signal → ADC → NAM A2-Lite → PWM out |
| Passthrough | Slow blink (1 Hz) | Guitar signal → ADC → PWM out (clean) |

Press & release **BOOTSEL** to toggle between NAM active and passthrough.

## Model

The default model is `example.nam` (a placeholder from oyama's repo).
To use a different A2 capture from [tone3000.com](https://tone3000.com):

```bash
# Download any A2 capture, then:
cmake -S . -B build -DNAM_MODEL=my_tone.nam -DCMAKE_BUILD_TYPE=Release
cmake --build build -j
```

Only A2-Lite (3-channel) models are supported — `nam2c` validates at build time.

## Source structure

| File | Role |
|------|------|
| `src/main.cpp` | ADC DMA, PWM timer, BOOTSEL toggle, main loop |
| `src/nam_fx.cpp` | Dual-core a2_fast NAM pipeline (adapted from oyama) |
| `src/nam_fx.h` | C-linkage interface for the NAM effect |
| `include/nam_model.h` | Embedded model interface (generated by nam2c) |
| `tools/nam2c.cpp` | Host tool: .nam → C weight array |
| `lib/NeuralAmpModelerCore` | NAM engine (submodule, oyama's fork) |

## USB CDC notes

- `pico_enable_stdio_usb(target 1)` in CMakeLists.txt — that's all that's needed
- **Do not add a custom `tusb_config.h`** — the SDK provides its own with the reset interface
- **Do not add `usb_descriptors.c`** — the SDK's `stdio_usb_descriptors.c` handles it
